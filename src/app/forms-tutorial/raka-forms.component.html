Angular forms are the same form concept as what we are using earlier in traditional approach. There are no any single changes in output because the design of form is still dependent upon the CSS styling. But there are huge changes in the term of development process.

Earlier few things were dependent upon the backend technologies regarding form validation but currently we can perform all kinds of validation from front end only using the power of Angular.

So Angular Forms are divided into two categories:

 

Difference between template driven form and reactive form?

With the template driven approach, you basically apply directives, such as ngModel, in your template. Based on these directives Angular will create formControl objects. This approach is good for building simple forms with basic validation (required, minlength, maxlength, etc)

With the reactive approach, you basically need to create new instances of the formControls and formcontrolgroups in your component. Reactive forms are also the best choice for building more complex forms and are better in case you have the intention to implement unit testing for your forms

Template Driven Form
  • Template Driven Form is suitable when we need to devlop the static form
  • Where the form field validation are fixed
  • Example are like Forgot password form, Login form
  • All Form validation is done on Template side.
  • We need to import the FormModule to work with Template Driven Form
Reactive Form
  • Reactive Form is more suitable when we need to develop the some complex form
  • Where form field is generating dynamically on the page validation is also more complex
  • Example are like - Big Complex Registration form where new form field is generating based scenario
  • All form validation is done in Class side. I mean Codes and the logic resides in the component class
  • And For working with Reactive Form we must need to include ReactiveForm Module from angular/forms
  • Here we have [formGroup] for Form tag. we need to associated and this will hold the one form
  • and FormControl is used for each form field

What is difference between ngModel and [ngModel] and [(ngModel)] ?

ngModel: This is reference of each form control, by using ngModel we can get all states of form control

[ngModel]: This is one-way data binding, we can display any data from class to template file using [ngModel]

[(ngModel)]: This is two-way data binding in angular. It is also called banana in box model. While working with [(ngModel)] we can pass data from view to component and component to view
