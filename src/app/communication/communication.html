We have few methods where we can establish the communication between components.

  • @Input() Decorator
  • @Output Decorator
  • Using services also
  • Behaviour Subject is also one of the method by using this we can pass message from different components by changing at one place.
  • @ViewChild() decorator is also one kind of option where you can make a communication.


    • What is the process of @Input() decorator?

    Well @Input() decorator is used to send information from parent component to child component. When we want to communicate between parent and child then we must need to use and import @Input() decorator.

    Steps to work with @Input() decorator

    • Firstly, we need to make whatever logic or information that what we need to send into child component
    • Caution: These logics need to implemented in parent component because we are sending data from to child
    • So far so good then we need to import child component selector in parent app and assign [property binding] with that logicâ€™s variable.

    For this entire process you must need to include the Input and Output from @angular/core library

    What is the process of @Output() decorator?

    Output decorator is the reverse order of Input decorator. When you want to send the data from child to parent component so this entire process is called @Output Decorator. This is Vice-Versa to each other.

    Steps to work with @Output() decorator

    • Firstly, we need to make whatever logic or information that what we need to send into child component
    • You need to use EventEmitter for this Because Child invoked the value by emit method.
    • Then in parent component element selector we need map the child event using () brackets and place the same event which you created into child component
    • Next - you copy the event name and use into parent component class do whatever you want.
