What is RestFul API
A RESTful API is an application programming interface (API) that uses HTTP requests and by using HTTP request you can perform some operation like -
  • GET
  • POST
  • PUT
  • DELETE data.

GET : By using this method we can retrieve the all data from API And we can bind this object into HTML as per our UI design.

POST: When you need to send any data from UI to backend so we can use POST method to send data in the form of object.

PUT: This method allows to update any existing record using one value and it will effect the backend data

DELETE: By using this HTTP method we can delete any existing record.

Dependency Injection in Angular

Dependency Injection is the coding pattern in which class receives its dependencies from external sources rather than creating itself.

  • As the name like Dependency injection - there is some object which are dependent to another project.
  • This is a way to create objects that depends upon other objects
  • Some component values are also dependent upon some other services. And service is not displaying any information until unless you requested something.
  • Make component reusable, maintainable and testable

 The Biggest benefit of using Dependency Injection is that - If you want to update something which is needed to all application then in that case you need to check at only one place and it will occured the all related places. You don't need to change everywhere.

 This is a good sign of Software Architecture. Because updating the execution mode from one place that is good.

Steps to integrate the Dependency Injection:

  • Firstly, we need to use services for making dependency injection
  • Use @Injectable() with provideIn:'root' meta data
  • Now your dependency injection setup is done and its ready to integrate.
  • Regarding this we can integrate Dependency injection in any component. we need to import the services file where we have alreay decorated the Dependency Injection.
  • Now we can use the [providers] inside the component meta data and later we can must have to attach with Constructor() method.
  • By this way you can refer the Dependency Injection you can perform any set of methods which is belogning from another component.

Observables Vs Promises?

Observables

Angular makes use of observables as an interface to handle a variety of common asynchronous operations.

Observables is the mechanism by using them we can communicate with server. We can retrieve the data and we can also send the data. I mean we can perform the any Rest ful API operations like HttpClient.get(),HttpClient.post(), HttpClient.put(), HttpClient.delete()

This is like the middle layer of API integration. Observables are communicating with from UI to API.

Observables are lazy, which means nothing will be happen until unless you are not subscribing to them.

Observables provides the methods such as map(), catch()

Observable is cancellable.

Observable returns a multiple values.

Promises

Whereas Promises are eager; which means as soon as a promise is created, the execution will take place and return the response either in success way or failed way. This is always returing the one value.

In observable you can pass multiple events at a time but with Promises it will not possible. Promise will handle the single event.
Promise provides the methods like then() and catch()

Promise is not cancellable.

Promise will return the single value whether it is Success or failed.

What is an interceptor in angular?

Interceptors provide a way to intercept for outgoing requests or incoming responses. They are very similar to the concept of middleware where its playing a role of handling & validiting the all https request.

Interceptors can be useful for features like caching and logging.
