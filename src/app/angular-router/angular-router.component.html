What is Wildcard Route in Angular?
Actually the Wildcard Route is basically used in Angular Application to handle the invalid URLs. Whenever the user is trying to enter some invalid URL or if you have deleted some existing URL from your application, then by default 404 page not found error page is displayed. This is more of avoiding the confusing while navigation to website.

What is Route Guards?

  • Route Guards concept is same as we know in our day-to-day life. Routes guards are always working as security layer to the any application.
  • If a user has authenticated then a user can attempt the next requested screen otherwise not. It means if you have a valid user credentials then you can visit a particular view.
  • Route Guards are always implemented as service that needs to be provided so we always make them @Injectable() while creation.
  • Route Guards are always returning true or false and telling you if your access is allowed or not.

We have some existing Route Guards:

CanActivate: while redirecting to other page CanDeactivate guard will ask you - are you really want to leave this page, once you confirm then you can leave!

CanActivateChild: This helps us to decide whether child routes can be activated or not. It is using for checking if user can visit child routes or not.

CanDeactivate: This help us to decide to whether route can be deactivated or not. Generally, this is using for giving warning to user, if user navigating from component So it is using to check to see if user can exit or not. This is like JavaScript confirm dialog.

Resolve: It is using for route data retrieval before route activation.

CanLoad: Checks to see if the user can navigate to the module which are lazy loaded.

What is Authentication and Authorization?  

Authentication is very important process in the system with respect to security. ... Authorization is the process of giving permission to the user to access certain resource in the system. Only the authenticated user can be authorised to access a resource.

Authentication

Actually Authentication is the process where a user is passing their login credentials to the server and server is examine with their login data and throughing a JWT (JSON Web Token) as return, And in this return we have a either True value or False. so this entire process is called a Authentication.

If JWT find correct user credentails then it can enable the page that you request so far.

Authorization

After login successfully, a different user have a different level of access. Some user may have access of everything and some user may have restricted level of access to resources. We can relate this concept by Subscribed and Unsubscribed users. As per this concept subscribed users can access everything about the application but unsubscribed users have not full access of the application.

Lazy Loading

Lazy Loading means on Demand Loading

Lazy Loading is one of the most excellent features from Angular. By using this feature our application performance will be high Because it does not load everything while loading the web page.

Earlier, in a traditional approach when main home page is loaded on the page at this time everything gets loaded and by this approach our application performance was not good.

Lazy loading modules speed up our applications startup time actually by this using approach when you need a page only that time your component will get loaded not all request. Means when you navigate the component only on that time your component gets loaded.

The concept of lazy loading is in angular requires us to format the application in a certain way. All the assets that are to be lazy loaded should be added to its own module. Lazy loading configuaration will be setup in the main app routing module file.

How to integrate Lazy Loading Module ?

  • Firstly we need create sub module on each and every component.
  • There will be two files: one is Module & another one is routing
  • We need to integrate the .forChild() routes inside routing file
  • Then this routing file need to integrate with module file.
  • Here we have App.routing file in root level and need to integrate this sub module in main path.
  • By this approach your all component will not be loaded. when you request any component on that time your component gets loaded.
