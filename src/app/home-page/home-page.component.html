What are the Entry Components ?

Entry Components are the components such as tiles and modals that you generate dynamically instead of with HTML selectors. You must declare the entry components to ensure that webpack does not remove them as unused code during the tree-shaking.Generally this is suitable for some login popup component.

What is tree shaking in angular?

Tree shaking is the ability to remove any code that we are not actually using in our application from the final bundle. It's one of the most effective techniques to reduce the footprint of an application. Basically this techniques removes the unwanted code which you are not using the application and accidently you mentioned it.

What is Transpiling in Angular?

Transpiling is the process of converting typescript into JavaScript, this is one of the most important concepts in angular, basically browser doesn’t understand typescript directly. Browser can only understand JavaScript CSS and HTML. So once our application development is done then we need to build using ng build and then all typescript code traspilled into JavaScript.

Why should ngOnInit be used ? If we have Constructor already

The main difference between constructor and ngOnInit is that ngOnInit is lifecycle hook and runs after constructor. Component interpolated template and input initial values aren't available in constructor, but they are available in ngOnInit . The practical difference is how ngOnInit affects how the code is structured.

The Constructor is the default method of class and its executed when the class is initiated and ensures the proper initializatoin of the field in the class.

Constructor creates the instance the another class by dependency

ngOnInit is the life cycle hook which is called by Angular. When you want to perform some action while component loaded in the browser so you need to bind with ngOnInit() life cycle hook.

And If you want to work with ngOnInit life cycle hook so you need to import the plugin from angular core package.

What are all the metadata properties of NgModule? And what are they used for?

@NgModule accepts a metadata object that tells Angular how to compile and launch the application. The properties are:

  • Declaration: The application's components, which belongs to the NgModule class. We must have to declare every component in an NgModule class. If we use a component without declaring it, we'll see a clear error message in the browser console, And this is array format value we can store multiple components which are required to the application.
  • Imports: Inside imports we can pass the modules which are reuired for the development process like Router Module, Forms Module, Common Module and other many modules.
  • Exports: Inside exports we are passing some configuration which are necessary for other component
  • Bootstrap: This is one of interesting meta of NgModule. Inside this we can pass the our first component which we need to show firstly while launching the application.

What is the use of polyfills.ts file in Angular

Polyfills in angular are few lines of code which make your application compatible for different browsers. The code we write is mostly written in ES6(New Features: Overview and Comparison) and is not compatible with IE or firefox and needs some environment setups before being able to be viewed or used in these browsers.

Polyfills.ts was provided by angular to help you do away with need to specifically setup everything.

Basically Polyfills is process of making our application browser compitable. Since we have different browsers and their version also so this is useful for compitable purpose nothing else
What the use of Zone.js in Angular ?

Zonejs is one of the core library used in Angularjs. Zone creates a wrapper around all asynchronous operations in the browser such as user interactions, HTTP, timers and any other changes that can cause changes in state.

By default, in an Angular app every task runs in the “Angular” Zone, which is called NgZone . There is only one Angular Zone and change detection is triggered exclusively for async operations which run in the NgZone
