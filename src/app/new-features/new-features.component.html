1) Hot Module Replacement
  • Hot Module Replacement is a mechanism that allows modules to be replaced without full browser refresh.
  • We don't need to configue this Because actually angular providing directly with the command line.

The command is like: ng serve --hmr This Features is generating the output without refreshing the web page. When you fire this command from command prompt -> It will enable the webpack js configuration and starts.

2) TypeScript 4.0 Script Support
  • Angular drop support of typescript 3.9
  • Now Angular 11 supports only 4.0
  • The main reason behind this to upgrade is: Speed up the build process
3) Webpack 5 Support
  • Angular 11 starts the experimental support of webpack 5
  • And if you want work with latest webpack 5 then you can update this version using package.jason file
  • By using this webpack 5 the projects bundle size will too smaller than other angular version
  • Since this is on Experimental mode So Angular Team also suggetsed - Don't use for the production envrionment. May be your bundle chunks not going smaller
4) Moving to the ESLint

Before knowing the ESLint we should to understand what is the Linting Process in Angular ?

Actually Linting is the process of improving the code quality and it is checking any potential errors in your code such as syntax errors, incorrectly spelled variable names, and many more. This can save your time and help you to write better code.

  • Angular 11 provides the ESLint. It means the ECMA Script Linting process
  • Earlier TSLint was popular. TSLint was one of the most popular linting tool among the developer for a long period of time.
  • But it was recently deprecated in version of Angular 11

Angular Architecture

What is decorator in angular
Decorator is one kind of mechanism or approach whose job to make decorating the class. Some task is like to maintain the Meta data and sometimes arranging the modules and components.

Here we have decorator which we are using to build the requirement:

  • @NgModules()
  • @Component()
  • @Directives
  • @Input()
  • @Output()
  • @injectable

Angular ng serve vs ng build

Usually, ng serves command is used to build, watch, and serve the application from local memory.

ng serve build the application in memory while ng build runs on the Hard Disk. So when you want to go for production deployment then you should to use ng-build.

After firing command like ng-build you can see one folder like Distt where you can see your components services directives all modules into Bundle format. and the size of project is too less for performance enhancement.

What does ng build --prod

ng build --prod flag compress your JS file, doing some action like below:

  • removes the unwanted material and comments
  • generating stable package for production envrionment and its ready to move
  • Doing minification of all files
  • Reducing folder size.
